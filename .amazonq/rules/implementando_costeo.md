# Regla: Implementando Costeo

## REGLA CRÍTICA: LEER Y ACTUALIZAR DOCUMENTACIÓN

### OBLIGATORIO:
- ✅ **ANTES de cualquier cambio**, leer `/docs/implementacion_costeo.md` completamente
- ✅ **DESPUÉS de cada cambio**, actualizar `/docs/implementacion_costeo.md` con:
  - Estado actual de implementación
  - Archivos modificados/creados
  - Funcionalidades completadas
  - Próximos pasos pendientes
  - Problemas encontrados y soluciones

### PROCESO OBLIGATORIO:

#### 1. ANTES DE IMPLEMENTAR
- Leer `/docs/implementacion_costeo.md` para entender estado actual
- Identificar qué componentes ya están implementados
- Verificar dependencias y prerequisitos
- Revisar patrones existentes en la aplicación

#### 2. DURANTE LA IMPLEMENTACIÓN
- Seguir exactamente los patrones establecidos en la aplicación
- Reutilizar componentes y estructuras existentes
- Mantener consistencia con el sistema de cache
- Usar mismos imports y convenciones

#### 3. DESPUÉS DE CADA CAMBIO
- Actualizar `/docs/implementacion_costeo.md` inmediatamente
- Documentar archivos creados/modificados
- Marcar tareas completadas
- Identificar próximos pasos
- Registrar cualquier problema o decisión de diseño

### ESTRUCTURA DEL DOCUMENTO:

```markdown
# Última actualización: [FECHA_HORA]

- [problemas encontrados y soluciones]
```

### PARA FUTURAS SESIONES:
**SIEMPRE comenzar leyendo `/docs/implementacion_costeo.md` para entender el estado actual antes de hacer cualquier cambio.**
