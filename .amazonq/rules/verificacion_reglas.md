# Regla: Verificación Obligatoria de Reglas

## REGLA CRÍTICA PARA TODAS LAS SESIONES

### ANTES DE CUALQUIER IMPLEMENTACIÓN:

1. **LEER TODAS LAS REGLAS** en `.amazonq/rules/` antes de escribir código
2. **VERIFICAR CUMPLIMIENTO** de cada regla aplicable al trabajo actual
3. **CONSULTAR MEMORY BANK** para patrones y lecciones aprendidas
4. **SEGUIR TROUBLESHOOTING** para evitar errores conocidos

### REGLAS CRÍTICAS A VERIFICAR SIEMPRE:

- **no_emojis.md**: No usar emojis excepto círculos y cuadrados de colores
- **never_placeholders.md**: Nunca usar implementaciones falsas o datos inventados
- **reutilizar_soluciones.md**: Revisar código existente antes de implementar
- **qa_testing_protocol.md**: Solo el usuario puede marcar problemas como resueltos
- **LEER_implementando_familias.md**: Leer y actualizar documentación de familias

### PROTOCOLO DE VERIFICACIÓN:

1. **Al iniciar sesión**: Leer reglas relevantes al trabajo solicitado
2. **Antes de cada implementación**: Verificar que no viole ninguna regla
3. **Durante desarrollo**: Consultar reglas si hay dudas
4. **Al finalizar**: Verificar que el resultado cumple todas las reglas

### CONSECUENCIAS DE NO SEGUIR REGLAS:

- **Pérdida de tiempo**: Rehacer trabajo que viola reglas establecidas
- **Errores repetidos**: Cometer errores ya documentados en troubleshooting
- **Inconsistencias**: Código que no sigue patrones establecidos
- **Frustración del usuario**: Tener que corregir violaciones obvias

### PARA FUTURAS SESIONES:

**SIEMPRE preguntar: "¿Qué reglas aplican a este trabajo?" antes de empezar cualquier implementación.**

## VERIFICACIÓN AUTOMÁTICA:

Antes de entregar cualquier código, verificar:
- ❌ ¿Hay emojis no permitidos?
- ❌ ¿Hay placeholders o datos inventados?
- ❌ ¿Se reutilizó código existente?
- ❌ ¿Se siguieron patrones establecidos?
- ❌ ¿Se consultó troubleshooting?

**Esta regla es OBLIGATORIA y debe seguirse en TODAS las sesiones futuras.**
